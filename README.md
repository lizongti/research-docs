# research-docs

调研文档专用仓库:技术调研、方案对比、选型结论与经验归档。

## 用途

集中存放各类**调研文档**,而非可运行的代码工程。典型内容包括:

- 技术/工具选型调研与对比
- 架构方案权衡与决策记录
- 第三方服务、库、协议的评估
- 踩坑记录与结论沉淀

## 目录约定

```
research-docs/
├── README.md
└── docs/                # 调研文档主目录(按主题/日期归档)
    └── <主题>/
        └── <yyyy-mm-dd>-<标题>.md
```

> 暂未预置 `docs/` 目录,新增第一篇调研时按上述约定创建即可。

## 写作建议

每篇调研文档建议包含以下要素:

| 部分 | 说明 |
| --- | --- |
| 背景 / 目标 | 为什么做这次调研,要回答什么问题 |
| 范围 | 调研覆盖与不覆盖的内容 |
| 对比 / 分析 | 候选方案、维度对比、证据 |
| 结论 | 明确的选型/决策结论 |
| 遗留 / 风险 | 未决问题、已知风险、后续动作 |

## 约定

- 文档以**简体中文**撰写;代码、路径、专有名词保持原样。
- 一篇文档聚焦一个调研主题,结论明确可追溯。
